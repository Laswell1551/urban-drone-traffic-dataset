# Sample

A small, self-consistent subset of the full dataset, committed to git so the
schema can be inspected and `examples/quickstart.py` can run without downloading
the ≈0.94 GB release.

**What it contains** — a fixed, seeded subset of **200 of the 6,911 drones**
(seed 42), in **4 scenarios** chosen to span the three OFAT axes:

| File | Condition | Demonstrates |
|---|---|---|
| `states/baseline__calm.parquet` | baseline, 20 m/s, 100 ft, calm | reference |
| `states/baseline__n10.parquet` | baseline + 10 m/s wind from north | wind / head–tail-wind split |
| `states/speed15__calm.parquet` | cruise speed 15 m/s, calm | cruise-speed axis |
| `states/alt200__calm.parquet` | cruise altitude 200 ft, calm | cruise-altitude axis |

Because the demand is shared across runs, a given `drone_id` is the same
origin–destination pair in every scenario, so the four files can be compared
drone-by-drone.

- `flights_sample.csv` — per-flight summaries for the sampled drones × 4 scenarios (800 rows).
- `hubs.csv` — the **full** 821-hub table (identical across runs).
- `scenarios.csv` — the **full** 30-run manifest (so the whole design is visible).

The schema is identical to the full release; see [`../DATA_DICTIONARY.md`](../DATA_DICTIONARY.md).
The sample is regenerated by the build script used for the release (seeded subset
of 200 drones over the four scenarios above).
